# Purpose

This document defines how approved carousel assets, carousel outlines, and carousel ready copy should be stored inside the Nayara Brand OS.

# How this document connects to the rest of the repository

Carousel_Index.md belongs inside 05_OUTPUTS because carousels are finished or reusable outputs. Carousel strategy should come from Foundation and Creation System files before any carousel is stored here.

# Core Principles

• Carousels must simplify complex ideas visually.
• Each carousel should have one clear argument.
• The first slide must create tension or clarity.
• The final slide must land the lesson or decision.
• Carousel copy must remain brand aligned and mobile readable.
• Approved carousel assets must not contain unsupported claims.

# Decision Framework

Before storing a carousel, ask:

1. Is the idea visual enough for carousel format?
2. Does each slide move the argument forward?
3. Is the hook strong and honest?
4. Is the visual hierarchy clear?
5. Is the copy aligned with Nayara's voice?
6. Are claims safe and supported?
7. Has the asset passed approval?

# Detailed Rules

Approved carousel categories:

• Capability gap explainers.
• AI readiness reframes.
• Decision frameworks.
• Skills risk breakdowns.
• Before and after thinking shifts.
• Customer safe pattern explanations.
• Personal brand framework visuals.

Rules:

Do not store raw Canva drafts here.

Do not store unapproved design experiments here.

Do not store carousels with weak slide logic.

Do not store carousels that rely on decoration instead of clarity.

Each approved carousel should include audience, objective, slide count, source document, and status.

Edge cases:

If a carousel is only a visual example, store it in 04_EXAMPLES.

If a carousel is a design draft, keep it outside the repository until approved.

If a carousel contains research, evidence must be documented before approval.

# Good Examples

Good example:

A five slide carousel explaining why AI readiness is a decision problem before it is a training problem.

Why this works:

It turns the core positioning into a clear visual argument.

Good example:

A carousel showing how skills gaps move from hidden friction to business risk.

Why this works:

It makes the brand philosophy easier to understand.

# Bad Examples

Bad example:

A carousel with beautiful design but no clear argument.

Why this fails:

Visual polish does not replace strategic clarity.

Bad example:

A carousel copied from a trend with Nayara's colours applied.

Why this fails:

It is not brand led.

# Common Mistakes

• Adding too many ideas.
• Making each slide too text heavy.
• Using the design as the point.
• Forgetting the final takeaway.
• Saving Canva drafts too early.
• Using claims without checking evidence.

# Anti Patterns

• Design first carousel creation.
• Trend copying.
• Slide clutter.
• Weak endings.
• Decorative visuals with no decision value.
• Unapproved design drafts in the repository.
