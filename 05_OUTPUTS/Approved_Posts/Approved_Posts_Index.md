# Purpose

This document defines how approved LinkedIn posts and other finished text posts should be stored inside the Nayara Brand OS.

# How this document connects to the rest of the repository

Approved_Posts_Index.md belongs inside 05_OUTPUTS because it stores finished public ready outputs. It should only contain or point to content that has passed creation and review standards.

# Core Principles

• Only approved posts belong here.
• Drafts belong outside the repository until approved.
• Each approved post should support Nayara's positioning.
• Each approved post should remain safe, accurate, and reusable.
• Posts should be organised so future AI sessions can learn from what has already been approved.

# Decision Framework

Before storing a post, ask:

1. Has the post passed the Creation Checklist?
2. Has the post passed the Review System?
3. Does it support the brand positioning?
4. Is it safe under Knowledge Boundaries?
5. Is it clearly labelled by theme or use case?
6. Is it worth reusing as a reference later?

# Detailed Rules

Approved post categories:

• AI readiness.
• Skills gaps.
• Workforce capability.
• Learning strategy.
• Technology adoption.
• Commercial risk.
• Personal professional lessons.
• Customer pattern observations with safe anonymisation.

Rules:

Do not store rough drafts here.

Do not store posts with unsupported statistics.

Do not store posts that expose confidential information.

Do not store posts that are off brand, even if well written.

Each approved post should include context, intended audience, content type, and status.

Edge cases:

If a post is approved but later feels outdated, move it to 99_ARCHIVE.

If a post is a strong example but not meant for publication, store it in 04_EXAMPLES.

If a post needs final review, do not store it here.

# Good Examples

Good example:

An approved post about AI readiness as a decision problem, labelled under AI readiness and workforce capability.

Why this works:

It is aligned, reusable, and easy to locate.

Good example:

An approved post about skills gaps showing up as delayed delivery.

Why this works:

It reinforces Nayara's core brand philosophy.

# Bad Examples

Bad example:

A promising post idea saved here before review.

Why this fails:

This folder is for approved outputs only.

Bad example:

A viral style post that does not support Nayara's positioning.

Why this fails:

Attention does not equal brand value.

# Common Mistakes

• Saving drafts too early.
• Keeping posts without labels.
• Storing strong writing that is strategically off brand.
• Leaving outdated posts active.
• Forgetting evidence safety after publication.

# Anti Patterns

• Draft archive disguised as approved posts.
• Viral post dumping.
• Unlabelled content storage.
• Posts without review evidence.
• Employer centred posts treated as personal brand assets.
