# Purpose

This document defines how reusable templates should be stored inside the Nayara Brand OS. Templates are finished structures that can be reused for posts, carousels, profile copy, newsletters, scripts, and future brand assets.

# How this document connects to the rest of the repository

Template_Index.md belongs inside 05_OUTPUTS because it stores reusable assets, not strategy or rules. It depends on the Foundation, Creation System, and Review System before any template can be added here.

# Core Principles

• Templates must be approved before being stored.
• Templates should make execution faster without weakening quality.
• Templates must not replace thinking.
• Templates must follow Nayara's positioning, voice, and knowledge boundaries.
• A template should be reusable, clear, and easy to adapt.

# Decision Framework

Before adding a template, ask:

1. What output does this template support?
2. Which brand rule does it depend on?
3. Is the structure reusable beyond one post?
4. Does it protect Nayara's voice?
5. Does it prevent generic content?
6. Has it passed review and approval?

# Detailed Rules

Approved template categories:

• LinkedIn post templates.
• Carousel templates.
• Profile copy templates.
• Newsletter templates.
• Comment templates.
• Content brief templates.
• Review templates.

Rules:

Do not store drafts here.

Do not store raw ideas here.

Do not store untested prompts here unless they have passed review.

Each template should include instructions, use case, structure, and quality checks.

Edge cases:

If a template is mostly an example, store it in 04_EXAMPLES instead.

If a template is a process, store the process in 02_CREATION_SYSTEM or 03_REVIEW_SYSTEM.

If a template becomes outdated, move it to 99_ARCHIVE only after a replacement exists or the reason is clear.

# Good Examples

Good example:

A LinkedIn reframe post template that includes hook, common belief, sharper reality, business consequence, and CTA.

Why this works:

It supports repeatable creation while staying aligned with the brand.

Good example:

A carousel template for explaining hidden capability risks in five slides.

Why this works:

It turns a brand idea into a reusable output format.

# Bad Examples

Bad example:

A random prompt that says write a viral LinkedIn post.

Why this fails:

It is not brand specific and has not passed review.

Bad example:

A half finished post saved as a template.

Why this fails:

Drafts do not belong in approved outputs.

# Common Mistakes

• Saving prompts as templates before testing them.
• Creating generic templates that ignore the brand voice.
• Treating templates as shortcuts around strategy.
• Storing examples and templates in the same place.
• Keeping outdated templates active.

# Anti Patterns

• Viral template dependency.
• Generic prompt banks.
• Unapproved templates.
• Template clutter.
• Drafts disguised as reusable assets.
• Templates that flatten Nayara's voice.
