# Purpose

This document defines the words, phrases, and language patterns that should appear across Nayara's content. It keeps the brand vocabulary consistent, specific, and recognisable.

# How this document connects to the rest of the repository

Vocabulary_Library.md supports hooks, posts, carousels, CTAs, stories, and profile copy. It should be used with Voice_Communication_System.md and Editorial_Principles.md before drafting content.

# Core Principles

• Use language that connects technology change to capability and business risk.
• Prefer concrete words over corporate abstractions.
• Make business consequences visible.
• Use human language without losing strategic weight.
• Avoid vocabulary that sounds like vendor marketing.

# Decision Framework

Before choosing language, ask:

1. Does this word make the idea clearer?
2. Does it sound like Nayara's brand?
3. Does it connect to decision making, capability, execution, or risk?
4. Would a busy leader understand it quickly?
5. Is there a simpler word available?

# Detailed Rules

Preferred vocabulary:

• Capability
• Decision
• Execution
• Risk
• Readiness
• Adoption
• Skills gap
• Workforce capability
• Commercial consequence
• Delivery friction
• Strategic priority
• Behaviour change
• Trust
• Visibility
• Trade off
• Business impact
• Talent constraint

Use with care:

• AI readiness
• Future of work
• Transformation
• Upskilling
• Reskilling
• Learning culture

Avoid unless necessary:

• Unlock
• Empower
• Leverage
• Game changer
• Seamless
• Robust
• Innovative solution
• Human potential
• Digital transformation journey

Edge cases:

If using a common buzzword, anchor it in a concrete consequence.

If using technical vocabulary, explain only what the reader needs to make the decision.

If using product language, keep the focus on the reader's problem.

# Good Examples

Good example:

Capability gaps do not stay inside learning teams. They show up in delivery, adoption, security, and customer experience.

Why this works:

It uses brand vocabulary and connects it to business consequences.

Good example:

AI readiness is a decision problem before it is a training problem.

Why this works:

It uses a familiar term but reframes it through Nayara's positioning.

# Bad Examples

Bad example:

Empower your workforce with innovative transformation solutions.

Why this fails:

It sounds generic and vendor led.

Bad example:

Unlock human potential through learning.

Why this fails:

It is vague and overused.

# Common Mistakes

• Overusing transformation language.
• Choosing impressive words instead of clear words.
• Using product vocabulary as brand vocabulary.
• Repeating the same phrase until it becomes stale.
• Using technical words without a decision reason.

# Anti Patterns

• Corporate filler.
• Buzzword stacking.
• Generic AI language.
• Product first phrasing.
• Abstract nouns without concrete outcomes.
• Language that hides the business consequence.
