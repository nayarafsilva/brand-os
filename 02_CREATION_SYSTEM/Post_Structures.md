# Purpose

This document defines approved content structures for Nayara's LinkedIn posts and other short form thought leadership. It prevents posts from becoming shapeless, overloaded, or generic.

# How this document connects to the rest of the repository

Post_Structures.md translates editorial principles into repeatable formats. It should be used after the idea, audience, and hook have been clarified.

# Core Principles

• Structure should make the idea easier to understand.
• One post should carry one core idea.
• Mobile readability matters.
• The structure should support the reader's decision, not just the writer's expression.
• A strong post has tension, development, and a useful landing.

# Decision Framework

Before choosing a structure, ask:

1. Is this educational, experiential, or tactical?
2. Does it need a story, a reframe, or a framework?
3. What should the reader think differently by the end?
4. Is the idea simple enough for a short post?
5. Should this be a carousel instead?

# Detailed Rules

Approved structures:

Reframe post:

1. Hook.
2. Common belief.
3. Sharper reality.
4. Why it matters.
5. Reader takeaway.

Pattern post:

1. Hook.
2. Observed pattern.
3. Why it happens.
4. Business consequence.
5. Practical question.

Story post:

1. Moment.
2. Tension.
3. Realisation.
4. Lesson.
5. Application.

Framework post:

1. Problem.
2. Three to five decision points.
3. Example.
4. Final principle.

Rules:

Use short paragraphs.

Avoid overloading the middle.

Do not add lists unless each point is distinct.

End with clarity, not noise.

Edge cases:

If the post has more than one idea, split it.

If the post explains a complex system, consider a carousel.

If the post is personal, use the Story post structure.

# Good Examples

Good example:

A reframe post about AI training that moves from workshop attendance to decision quality.

Why this works:

It turns a familiar topic into a sharper brand aligned idea.

Good example:

A pattern post about skills gaps showing up as delayed delivery.

Why this works:

It makes an abstract problem visible.

# Bad Examples

Bad example:

A post with AI, career advice, sales lessons, and learning strategy all together.

Why this fails:

It has too many ideas.

Bad example:

A long paragraph with no clear landing.

Why this fails:

It is hard to read and weak on mobile.

# Common Mistakes

• Choosing a structure before clarifying the idea.
• Adding too many points.
• Ending with a weak CTA.
• Using a list when a story would be stronger.
• Using a story when a reframe would be clearer.

# Anti Patterns

• Shapeless posts.
• Overloaded posts.
• Generic listicles.
• Carousels squeezed into text posts.
• Emotional stories without a useful landing.
• Frameworks with no decision value.
