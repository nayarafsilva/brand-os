# Purpose

Photography OS is the source of truth for creating, reviewing, approving, storing, and reusing photographs of Nayara Fernandes for her personal brand.

It exists to create a consistent photographic identity that can support social media posts, LinkedIn content, carousels, website assets, speaker materials, interviews, podcast promotion, and future brand channels without requiring a new professional photoshoot for every use case.

The system must protect Nayara's identity while allowing controlled changes to pose, expression, crop, scene, styling, and approved wardrobe.

# How this document connects to the rest of the repository

Photography OS is a subsystem of the Nayara Brand OS.

The Brand OS defines Nayara's positioning, personality, audience, voice, and broader brand standards.

Photography OS translates those standards into photographic decisions.

# Core Principles

• Identity before aesthetics.
• Recognition before novelty.
• Authority before fashion.
• Proximity without loss of credibility.
• Natural human detail before artificial perfection.
• Consistency before experimentation.
• Believability before visual spectacle.
• Approved brand colours before trend colours.
• Realistic professional photography before illustration or obvious AI styling.
• Reusable assets before unnecessary new generation.
• External inspiration informs pose or composition only. It never replaces Nayara's identity.
• Every approved image must look as though it belongs to the same personal brand, even when outfits and scenes differ.

# Decision Framework

Before creating a photograph, answer these questions:

1. What channel and content purpose will the image support?
2. What should the viewer feel first?
3. Does the image need authority, proximity, or a deliberate balance of both?
4. Which approved outfit and scene best support that purpose?
5. Is a suitable approved image already available?
6. Which reference image defines Nayara's face for this generation?
7. Which elements may change?
8. Which elements must remain locked?
9. Does the final output look like Nayara rather than a similar woman?
10. Would the image be credible as a real professional photoshoot?

If the image does not have a clear use case, do not create it.

If identity accuracy is weak, reject the image regardless of how attractive the composition appears.

10_ASSET_LIBRARY

Purpose:

Maintains a searchable record of approved images, intended uses, outfit IDs, scene IDs, crops, and status.

Image files should use clear descriptive names and never rely on random tool generated filenames as the permanent asset name.

Approval rule:

No image or document enters an approved Photography OS folder until it is:

• Explicitly approved.
• Compliant with the Nayara Brand OS Repository Standards.
• Compliant with Photography OS non negotiable rules.
• Checked against the relevant reference images.
• Checked for identity accuracy.
• Checked for AI artefacts.
• Assigned a clear purpose and asset status.

Edge cases:

If a reference image contains clothing outside the approved palette, retain the pose or composition only and restyle the clothing.

If a generated image has the correct outfit but the wrong face, reject it.

If a generated image has a strong face match but an incorrect accessory, correct only the accessory when possible.

If a requested image conflicts with a permanent rule, the permanent rule wins unless Nayara explicitly changes the rule and the repository is updated.

If text must appear inside an image, generate the photograph first and add final typography in a design tool when exact spelling, placement, or brand consistency matters.

# Good Examples

Good example:

Use an external studio portrait as inspiration for a seated pose, but apply Nayara's approved face, dark brown to black hair, warm skin tone, gold jewellery, and approved neutral outfit.

Why this works:

The inspiration controls composition without replacing identity.

Good example:

Create several crops from one approved studio photograph for LinkedIn posts, carousel covers, and profile assets.

Why this works:

It increases asset value without creating unnecessary identity variation.

Good example:

Use a clean handwritten text treatment around an approved portrait, but add the final words in Canva or another controlled design tool.

Why this works:

It combines proximity with reliable typography.

# Bad Examples

Bad example:

Copy the model's face, hair colour, and clothing because the reference image looks polished.

Why this fails:

The reference becomes the identity and Nayara disappears.

Bad example:

Approve an attractive image even though the face does not look like Nayara.

Why this fails:

Aesthetic quality cannot compensate for identity drift.

Bad example:

Generate random fashionable outfits to create variety.

Why this fails:

Variety without a controlled system weakens recognition.

# Common Mistakes

• Treating one generated face as permanently accurate without comparing it to real references.
• Allowing inspiration images to alter ethnicity, face structure, hair colour, or age.
• Building too many scenes before locking identity.
• Creating visual variety through unrelated clothing.
• Approving images because they are flattering rather than accurate.
• Mixing approved photographs with external inspiration.
• Repeating full prompts instead of loading approved system documents.
• Ignoring hands, jewellery, watch, teeth, or facial asymmetry during review.
• Using generated text as final artwork when exact typography matters.
• Saving assets without names, status, use cases, or approval notes.

# Anti Patterns

• Generic AI businesswoman imagery.
• Luxury influencer styling.
• Corporate stock photography.
• Identity drift between scenes.
• Over retouched skin.
• Unrealistic teeth or eyes.
• Excessive hair volume.
• Unapproved highlights or lighter hair.
• Random colour palettes.
• Black separates used outside an approved exception.
• Mixed metal jewellery.
• Trend led sets that date quickly.
• Props that distract from Nayara.
• Images with no content purpose.
